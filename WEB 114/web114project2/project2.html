1. At the end of the Browser environment lesson, the author makes this comment "Now we’ll get down to learning DOM, because the document plays the central role in the UI."

"Now we’ll get down to learning DOM, because the document plays the central role in the UI." - Author

The DOM is the document object model. The document is the "entry point" that we use to access the different document object models within the page. The UI is the user interface. It is basically all the visual elements that the user sees when they visit a website. In JavaScript, for example, we can use a statement like "document.body.style.background = 'red';" to change the body element color of the website.

So let's review some basic JavaScript terminology as it relates to its connection with a web page.

3a. What is the document? And what is the document object?

As said earlier, The document is the entry point of the DOM. It is the root of the DOM tree. The document objects are the different objects within the HTML file. An HTML file contains different elements. (title, head, body, etc.) These are the different objects within the HTML file.

3b. What operator do you use to access the methods and properties of an object?

To access the methods and properties of an object, you would use the dot operator. (.) For example, to access the body element of an HTML file, you would write something like this:

document.body.style.background = 'red';

3c. What characters always follow a method?

Characters that always follow a method are parenthesis (). This looks like this:

console.log("HEHEHEHEHE");

3d. What, in general, does a method of an object do?

A method of an object is similar to a function within a class in any other language. I am familiar with C++ so I will use a C++ example. An object is similar to a class in C++. Within that class, you may have functions that are associated with that specific class.

3e. What, in general, does a property of an object do?

A property of an object is similar to variables stored within a class. Another way we could see similarities is within SQL where you have a table. You can make the comparison by imagining the different keys within the table as properties within an object.
