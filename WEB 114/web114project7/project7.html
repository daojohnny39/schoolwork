Answer I

The first one is using curly braces when writing functions, if statements, and loops.

if (n < 0)
   alert(`Power ${n} is not supported`);

This kind of syntax can bring a lot of errors in the future when the writer decides to add more lines of code. Let's look at the "correct" syntax for that statement:

if (n < 0) {    // <-- opening bracket
   alert(`Power ${n} is not supported`);
}   // <-- closing bracket

Adding curly braces makes sure that the code will run accordingly to how the writer wants it to run. It also makes it easier to distinguish the scope of the code whether it's a function, if statement, or loop.

Answer II

Another practice commonly used is splitting long lines of code or text into separate lines to improve readability. Let's look at an example:

let str = ` ECMA International's TC39 is a group of JavaScript developers, implementers, academics, and more, collaborating with the community to maintain and evolve the definition of JavaScript.
`;

As you can see, it's pretty hard to distinguish the text from the code. Let's look at an example where the text is split into separate lines:

let str = `   // <-- break line here
   ECMA International's TC39 is a group of JavaScript developers,
   implementers, academics, and more, collaborating with the community
   to maintain and evolve the definition of JavaScript.
`;

As you can see, that text was a lot easier to read as well as the code. This is a good practice to use when there are long lines of text or code. (Especially on those if statements with multiple conditions)

Answer III

Another thing that is good practice is adding enough whitespace between lines of code. Let's look at an example:

function pow(x, n) {
   let result = 1;
   for (let i = 0; i < n; i++) {
      result *= x;
   }
   return result;
}

This piece of code will run correctly. However, the issue comes when you are trying to debug the code or just reading it in general. Let's look at another example where sufficient whitespace is added:

function pow(x, n) {

   let result = 1;

   for (let i = 0; i < n; i++) {
      result *= x;
   }

   return result;

}

As you can see, the whitespace allows your eyes to distinguish the different parts within the code. This is good general practice to implement in your code to allow other people to read it easier.
