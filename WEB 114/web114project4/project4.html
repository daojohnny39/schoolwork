Creating variables and using alert(), confirm(), and prompt() to interact with the user, using if-then-else structures

1. alert(), confirm(), and prompt() are methods of which object?

Answer

The alert(), confirm(), and prompt() are methods of the window object.

2. Why can we use the alert(), confirm(), and prompt() methods without preceding them with the object. (object dot) syntax??

Answer

The reason why we are able to use alert(), confirm(), and prompt() without adding "window." is because usually it is assumed that when using these functions, you are calling them from the window object. This can be redefined and alert(), confirm(), and prompt() will not work the same after.

3. How do we distinguish variable names from strings in JavaScript?

Answer

You can distinguish the difference of a JavaScript variable name from a string in JavaScript by looking at the syntax around. Variables in JavaScript usually start with a lower-case letter, an underscore, or a dollar sign. For a string, it is usually surrounded in quotes.

var myAge = 10; // notice that "myAge" starts with a lowercase letter
var myAge = "ten"; // "ten" is surrounded in quotes

Answer

The point of learning these popup boxes is that it can help us in terms of debugging. It also makes learning the language a lot easier since these popup functions are limited in terms of customization.
