2. Why is the JavaScript textContent property safer than innerHTML?

Answer

Because textContent will convert all the HTML tags to their "ignore characters" whereas using innerHTML will cause the HTML tags to actually visually display. It is similar in a way to linux commands having "\" as an ignore character. Except, HTML doesn't use "ignore characters".

elem1.innerHTML = name;

This would display: Winnie-the-Pooh!

elem2.textContent = name;

This would display: <b>Winnie-the-Pooh!</b>

3. Every HTML element you write as a developer becomes a corresponding DOM object. And remember -- an object is just a fancy storage container. An object contains two things: properties and methods. And each property of the document object may contain a single value or it could contain another object! For example body is a property of the document object. And what does body contain?

Another object of properties and methods. So you might write a statement such as this.

document.body.id= "light"; // This is in the JavaScript

Which does this to the DOM:

<body id="light"> <! -- this is the change made in the HTML -->

So in the JavaScript, document is an object that connects to the entire web page. It contains many methods and properties one of which is named body.
And the body property contains another object which contains many methods and properties, one of which is named id.
And the id property can be assigned to a value which changes the <body> element in the HTML.
Now based on this example, what do you notice about JavaScript properties and their corresponding HTML attribute names?

Answer

The body would contain the id attribute. It seems like in JavaScript, the document is very similar to a class in C++ with its own variables and functions. The different HTML tag properties would be fed as class variables or class functions. For example, in order to access the id property of the body, you would have to treat the body as almost a class within a class, calling the document "class" first, then calling the "body" class (in this example, inside the document class), then accessing the "id" variable inside the body class. The coding statement would look like this:

document.body.id = "light";

4. Why do we rarely, if ever use document.write()?

Answer

When you use document.write() to try to debug, the whole page gets overwritten to whatever you decide to input into the parameter. To prevent this, we use console.log() instead so that when we're debugging, instead of the whole document being overwritten, we can just check the console to see if everything is working the way it should.

5. What is a JavaScript method? Define and provide two examples.

Answer

JavaScript methods are functions that are associated with an object.

Let's turn strings into all uppercase for example:

let str = "Hello World!";
let res = str.toUpperCase(); // outputs: "HELLO WORLD!"

Let's look at another where we create a div element:

let div = document.createElement("div");
document.body.appendChild(div);

The first line creates a div element and the second line appends the div element to the body of the HTML document.

6. What is a JavaScript property? Define and provide two examples.

Answer

A JavaScript property is very similar to variables in C++. They are key-value pairs within an object, or variables within a class.

let person {
    firstName: "John",
    lastName: "Doe",
    age: 50,
    eyeColor: "blue"
};

We can look at an example for arrays as well:

let cars = ["Toyota", "Honda", "Hyundai"];
console.log(cars.length); // outputs: 3

8. What is a CSS selector? Define and provide two examples.

Answer

A CSS selector is method where certain patterns of syntax will allow you to select certain elements in the HTML document.

For example, we can change the color of <p> elements to red:

p {
    color: red;
}

We can also select classes within the HTML document by using the "." symbol:

.question-within-question {
    color: red;
}

9. What is a CSS property? Define and provide two examples.

Answer

A CSS property is a key-value pair in CSS. It is the "color" property that we used in the example above.

Let's take a look at some other properties that we can use:

h1 {
    font-size: 1.2em;
    text-align: center;
    text-decoration: underline;
}

In this example, we are changing the font size to 1.2em, center aligning any texts within h1, and underlining the text.

10. What is a CSS declaration? Define and provide two examples.

Answer

Once again, we can look at the examples above to see what a CSS declaration is.

h1 {
    font-size: 1.2em;
    text-align: center;
    text-decoration: underline;
}

For each CSS property, we are declaring a value for it. Font-size is declared to 1.2em, text-align is declared to center, and text-decoration is declared to underline.
